# ProgrammingPatterns

Singleton: Garantiza que una clase tenga solo una instancia y ofrece acceso global a ella.

Factory Method: Define una interfaz para la creación de objetos, permitiendo que las subclases modifiquen el tipo de objeto generado.

Abstract Factory: Suministra una interfaz para producir familias de objetos relacionados sin especificar las clases concretas.

Prototype: Se utiliza para instanciar nuevos objetos mediante la clonación de un prototipo existente.

Builder: Separa la construcción de un objeto complejo de su representación, permitiendo la creación de diversas representaciones.

Adapter: Facilita la colaboración entre interfaces incompatibles, transformando la interfaz de una clase en otra compatible con las expectativas del cliente.

Composite: Ensambla objetos en estructuras jerárquicas de árbol para representar totalidades y partes, permitiendo la interacción homogénea con objetos individuales y composiciones.

Decorator: Autoriza la adición dinámica de funcionalidades a objetos sin alterar su estructura original.
